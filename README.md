# Лабораторная работа 2

## Подключение Jenkins и сборка проекта

### Цель работы

Настроить систему непрерывной интеграции (CI) с 
использованием Jenkins.
Необходимо подключить созданный ранее сервер как 
Jenkins Node, создать задачу сборки проекта и 
настроить сохранение артефактов.

**Задачи**

В рамках лабораторной работы необходимо:

* подключить сервер, созданный в ЛР1, как **Jenkins Node**;
* создать Jenkins job для сборки собственного проекта;
* настроить выполнение **сборки** на подключенном сервере;
* настроить **архивацию артефактов** сборки.

## Выполнение работы

В качестве проекта используется приложение для 
классификации музыкальных жанров. Проект состоит 
из двух сервисов:

* **API сервис** (FastAPI), который загружает обученную модель и выполняет предсказание;
* **Telegram-бот**, который принимает аудио от пользователя и отправляет его в API для классификации.

На сервере из ЛР1 были установлены необходимые инструменты:

* Java
* Docker
* Docker Compose

После этого сервер был подключён к Jenkins как **Node**.  
Подключение выполнялось через веб-интерфейс Jenkins с использованием **SSH credentials**.

В Jenkins был создан pipeline, который:

1. клонирует репозиторий проекта;
2. собирает Docker-образы сервисов (api and bot);
3. публикует собранные образы в **DockerHub**.
