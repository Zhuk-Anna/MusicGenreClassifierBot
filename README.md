# Лабораторная работа 5 и 6

## Создание инфраструктуры с Terraform и настройка с Ansible. Построение CI/CD Pipeline

### Цель работы

Освоить использование инструментов **Infrastructure as Code** для автоматического 
создания и настройки инфраструктуры. В рамках работы необходимо реализовать 
создание серверов с помощью **Terraform** и автоматическую настройку программного 
обеспечения с использованием **Ansible**.

Объединить процессы сборки приложения, создания инфраструктуры и деплоймента 
в единый автоматизированный **CI/CD pipeline** с использованием Jenkins.

### Задачи

В рамках лабораторной работы необходимо:

* реализовать создание инфраструктуры с использованием **Terraform**;
* описать параметры создаваемых ресурсов (сервер, сеть, ключи доступа);
* автоматически создать виртуальную машину в облачной инфраструктуре;
* настроить установку системного и прикладного программного обеспечения 
  с использованием **Ansible**;
* объединить этапы сборки приложения, создания инфраструктуры и деплоймента 
  в один Jenkins pipeline;

---
### Настройка окружения Jenkins Node

На Jenkins Node были установлены следующие инструменты:

* **Terraform** через зеркало от Яндекс.Облака для работы с OpenStack;
* **Ansible** для автоматической настройки серверов и деплоймента Docker-контейнеров;

Также настроены Jenkins credentials для доступа к:

* Docker Hub (логин и пароль)  
* OpenStack (пароль пользователя)  
* Telegram Bot Token (для запуска бота)  
* SSH приватный ключ (для подключения к создаваемым серверам)

---

### Файлы Terraform и их назначение

В репозитории инфраструктуры присутствуют следующие файлы:

* `main.tf` – описание ресурсов виртуальной машины (сервер, сеть, том для загрузки образа). Используется `Boot from Volume`, то есть создаётся отдельный том для загрузки образа, который подключается как блоковое устройство и автоматически удаляется при удалении VM.  
* `variables.tf` – переменные, используемые при создании ресурсов (flavor, сеть, образ, ключи и размер тома).  
* `outputs.tf` – вывод IP-адреса созданного сервера для последующего использования в Ansible.  
* `provider.tf` – конфигурация провайдера OpenStack и требований к версии Terraform.

---

### Основные шаги Ansible playbook

Playbook `playbook.yml` выполняет следующие действия:

1. **Запуск Terraform** для создания инфраструктуры (VM) через модуль `community.general.terraform`.  
2. **Dynamic Inventory** – добавление IP нового сервера в группу `music_servers` для последующих тасков.  
3. **Установка Docker и Docker Compose** на созданном сервере.  
4. **Логин в Docker Hub** для загрузки образов.  
5. **Копирование `docker-compose.yml`** на сервер.  
6. **Запуск Docker Compose** для поднятия контейнеров API и Telegram Bot.  

Playbook использует **dynamic inventory**, что позволяет автоматически управлять вновь созданными серверами.

---

### Шаги Jenkins Pipeline

Jenkins pipeline объединяет все этапы в один CI/CD процесс:

1. **Check server** – проверка доступности инструментов на Jenkins Node (`docker`, `terraform`, `ansible`).  
2. **Build API and Bot Images** – сборка Docker образов для API приложения и для Telegram бота.
3. **Login to Docker Hub and push Images** – авторизация и загрузка образов с тегами `latest` и `${APP_VERSION}.${BUILD_NUMBER}`. 
   * `latest` – всегда указывает на самую свежую сборку для удобного быстрого деплоя;
   * `${APP_VERSION}.${BUILD_NUMBER}` – уникальный тег конкретной сборки, позволяющий откатиться или воспроизвести точную версию приложения.
4. **Create Infrastructure, Configure, Deploy** – запуск Ansible playbook, который вызывает Terraform для создания VM, устанавливает Docker, Compose, настраивает окружение и поднимает контейнеры.
