# Лабораторная работа 7

## Развертывание приложения в Kubernetes (Yandex Cloud)

### Цель работы

Освоить развертывание контейнеризированного приложения 
в Kubernetes-кластере, настроить подключение к облачному 
кластеру и выполнить деплой микросервисного приложения 
через YAML-манифесты.

### Задачи

В рамках лабораторной работы необходимо:
1. Подключиться к существующему Kubernetes-кластеру в Yandex Cloud.
2. Настроить локальный доступ к кластеру через `kubectl`.
3. Разработать YAML-манифесты для развертывания:
   - API модели
   - Telegram-бота
4. Выполнить деплой приложения в кластер.
5. Проверить корректность работы контейнеров.


### Выполнение 

**Были установлены и настроены:**

- `yc CLI` — для работы с Yandex Cloud
- `kubectl` — для управления Kubernetes
- Настроено подключение к кластеру
- Сформирован конфигурационный файл `kubeconfig`

Команда подключения к кластеру:

```bash
yc managed-kubernetes cluster get-credentials --id <cluster-id> --external
```

**Проверка подключения:**

```bash
kubectl cluster-info
kubectl get nodes
kubectl get pods
```
Кластер успешно подключен, узлы находятся в состоянии `Ready`. 
После деплоя контейнеры были успешно запущены и перешли 
в состояние `Running`.

**Деплой:**

Для развертывания приложения были созданы три YAML-файла: 
* для API (Deployment)
* для Bot (Deployment)
* для Service (ClusterIP) нужного для взаимодействия компонентов. 

API и бот запускаются как отдельные контейнеры из образов, 
размещённых в DockerHub.

Для безопасного хранения токена Telegram использован 
**Kubernetes Secret**, который подключается к контейнеру 
через `secretKeyRef`, что исключает хранение чувствительных 
данных в репозитории.

```bash
kubectl create secret generic <secret-name> --from-literal=<secret-key>=<secret>
```

Развертывание выполняется командой `kubectl apply -f k8s/`, после чего Kubernetes автоматически создаёт 
и управляет pod'ами. Настроены проверки состояния 
(liveness/readiness), обеспечивающие автоматический перезапуск
при сбоях.

Удаление выполняется командой `kubectl delete -f k8s/`. 

Просмотр логов выполняется командой `kubectl logs <pod name>`.
