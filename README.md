# Лабораторная работа 4

## Деплой приложения через Jenkins

### Цель работы

Автоматизировать процесс развёртывания приложения на созданной инфраструктуре с использованием Jenkins.  
Необходимо передавать артефакты сборки на сервер, запускать приложение и оформлять процесс деплоймента в виде Jenkins pipeline.

### Задачи

В рамках лабораторной работы необходимо:

* передать Docker-образы на созданный сервер;
* выполнить запуск сервисов приложения;
* оформить процесс деплоймента в Jenkins pipeline;

---
Pipeline реализован с использованием следующих шагов:

1. **Получение IP сервера**:  
   Через OpenStack CLI pipeline получает IP сервера из output параметров стека (`server_ip`).

2. **Передача конфигурации**:  
   С помощью `scp` на сервер копируется `docker-compose.yml`.

3. **Авторизация в DockerHub**:  
   Jenkins использует credentials для логина. Образы скачиваются из DockerHub и обновляются при необходимости.

4. **Запуск контейнеров через Docker Compose**.

Для API сервиса добавлен `healthcheck`. Контейнер 
бота зависит от API и запускается только после 
того, как API становится `healthy`.
